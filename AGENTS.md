# BookmarkPrimitive Working Guide

## Purpose
BookmarkPrimitive owns named anchors, auto-generated bookmark regeneration, cross-reference resolution, and observable bookmark mutations.

## Key Directories
- `Sources/BookmarkPrimitive`: Bookmark models, cross references, and `BookmarkStore`.
- `Tests/BookmarkPrimitiveTests`: Model and store behavior tests.

## Architecture Rules
- Keep bookmark anchors independent from rich text or document packages. Consumers bridge `BookmarkAnchor.contentID` and `offset`.
- Use `BookmarkPositionResolver` as the only seam for page numbers and above/below comparisons.
- Route mutations through `BookmarkStore` so observers receive `BookmarkMutation` events.
- Preserve auto-generated bookmarks during regeneration by replacing only `isAutoGenerated` entries.

## Testing
- Run `swift test` before committing.
- Add `BookmarkStoreTests` for mutation, regeneration, and cross-reference behavior.
- Add model Codable/equality coverage in `ModelTests`.
